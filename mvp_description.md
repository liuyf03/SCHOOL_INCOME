# School Income Explorer — MVP Plan (Washington State)

## Locked scope

- **What we estimate:** the income distribution of _households with school-aged children_ (own children under 18) living inside each public school's attendance zone. Not the enrolled families — the zoned families. We'll surface this caveat in the UI.
- **Geography:** Washington state only (state FIPS `53`). ~2,400 public schools, ~5,300 census block groups.
- **Frontend:** a single static webpage with a school-name search bar and a results panel. No map.
- **Backend:** none. The site loads a precomputed static data file generated by an offline pipeline.

## Step 1 — Acquire raw data (one-time, WA-only subsets)

Pull these once and check them into a `data/raw/` folder:

1. **NCES SABS** (School Attendance Boundary Survey) shapefile, latest release. Filter to `STATEFP = '53'`. This gives attendance-zone polygons. Coverage in WA is partial — record which districts are missing for the "known gaps" disclosure.
2. **TIGER/Line block groups** for WA only (one shapefile, state FIPS 53). Match the vintage year to the ACS release.
3. **TIGER/Line census blocks** for WA — needed as the population-weighting layer for dasymetric interpolation.
4. **ACS 5-year estimates** at the block-group level via the Census API, WA only:
   - `B19131` — family income in the past 12 months by family type by presence of own children under 18. This is the table that filters to families with kids.
   - `B11005` — households by presence of children under 18 (sanity check + denominator).
   - `B19013` — median household income (for the zone-level summary stat).
5. **NCES Common Core of Data (CCD)** school directory, filter to WA. Provides `NCESSCH` ID, school name, grade range, address, district, and FRL counts (for validation in step 5).

## Step 2 — Build the offline pipeline

A single Python script (`build_dataset.py`) that runs end-to-end:

1. Load all four geospatial layers into GeoPandas in a common projected CRS (EPSG:2927, WA State Plane South, or EPSG:3857 if you don't care about area accuracy).
2. For each school attendance polygon, find intersecting block groups via spatial index.
3. Use `tobler.area_weighted.area_interpolate` with the census-block population layer as the auxiliary weight. Interpolate the `B19131` bracket counts and `B11005` "with own children" counts from block group → school zone.
4. From the interpolated bracket counts, compute per-school summary fields: estimated median family-with-children income, share under $35k, share over $150k, total estimated families with children, and the full bracket histogram.
5. Join to the CCD directory on `NCESSCH` to attach school name, district, grades, address.
6. Write the result as a single `schools_wa.json` file (or NDJSON), keyed by `NCESSCH`. For ~2,400 schools this should be well under 5 MB — small enough to ship with the site.

Also emit a separate `search_index.json` — just `{nces_id, name, district, city}` for fast client-side autocomplete.

## Step 3 — Validate

Before declaring the dataset done:

1. **FRL sanity check:** compute Pearson correlation between your "share of families with children under $35k" and the school's FRL eligibility rate from CCD. Expect r > 0.6. If it's near zero, the spatial join is broken.
2. **Coverage report:** count schools where SABS polygon was missing or where interpolated household count is < 50 (likely a bad polygon). Surface these as "low confidence" in the UI rather than hiding them.
3. **Spot-check 5 schools manually:** pick one urban (Seattle), one suburban (Bellevue), one rural (somewhere in eastern WA), one tribal area, and one with known wealth concentration (Mercer Island). Eyeball whether the medians match local intuition.

## Step 4 — Frontend (static site)

A single-page app, no build step required if you don't want one:

- `index.html` + `app.js` + `style.css`.
- On load, fetch `search_index.json` (small) and build a client-side fuzzy-search index using Fuse.js or MiniSearch.
- Search bar at the top with autocomplete dropdown.
- On selection, lazy-fetch that school's record from `schools_wa.json` (or split into per-school JSON files if you want smaller initial payload).
- Results panel shows: school name, district, grades, estimated median family income, a histogram of the income brackets (Chart.js), and a comparison line for WA state median.
- Persistent disclaimer block: "These figures estimate households with children _zoned_ to this school based on Census ACS data. They do not reflect enrolled families and exclude private-school and school-choice effects."

Host on GitHub Pages, Netlify, or Cloudflare Pages — all free, all serve static files.

## Step 5 — Repository layout

```
school-income-mvp/
  data/

    download.py       # fetches all raw inputs

    validate.py       # FRL correlation + coverage report
  site/
    index.html
    app.js
    style.css
  README.md
```

## Step 6 — Tech stack (MVP)

- **Pipeline:** Python 3.11, GeoPandas, Shapely, `tobler` (PySAL), `census` or raw `requests` against the Census API, pandas.
- **Frontend:** vanilla JS or a tiny framework (Alpine.js, Preact); Chart.js; Fuse.js or MiniSearch for search.
- **Hosting:** GitHub Pages.

No database, no server, no auth, no API — everything is static files.

## Step 7 — Extension paths (post-MVP, not built now)

The MVP is structured so each of these is an additive change, not a rewrite:

1. **More states.** The pipeline is parameterized by state FIPS — running for all 50 states produces ~100k schools. At that point switch the static blob to per-school JSON files served from a CDN, or move to a real backend (FastAPI + DuckDB/PostGIS).
2. **Map view.** Add a MapLibre layer rendering the SABS polygons, colored by median income. The polygon GeoJSON already exists in the pipeline output.
3. **Address lookup.** Add a "find my school" address input that geocodes via the Census Geocoder and point-in-polygon lookup against SABS.
4. **Refresh cadence.** Wrap `build_dataset.py` in a GitHub Action that reruns annually when ACS publishes the new 5-year release.
5. **Margins of error.** Pull the ACS MOE columns alongside the estimates and propagate them through the interpolation. Display as error bars on the histogram.
6. **Enrolled-family estimates.** For districts that publish enrollment-by-zone or transfer data, blend that in to move from "zoned" to "enrolled" estimates.
7. **Side-by-side comparison.** Pin two or three schools and compare distributions in the same chart.
8. **Trend over time.** Recompute against past ACS vintages to show how a zone has shifted.

## Step 8 — Estimated effort

- Pipeline (download + spatial join + validation): 2–3 evenings.
- Static site: 1 evening.
- Validation + writing the disclaimers carefully: 1 evening.

Total: roughly a long weekend for a working WA-only site.
